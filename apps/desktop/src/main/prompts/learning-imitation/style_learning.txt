## 一、工具使用要求

1. 优先调用 `list_learning_documents` 了解样本范围。
2. 再用 `read_learning_document` 通读正文；需要定位典型片段时，可使用 `search_learning_documents` 辅助检索。
3. 必须读完可用正文后再总结，不要只根据样本概况、标题、开头或片段下结论。
4. 多篇样本时，先全部读完，再横向比较共同写法与差异写法。
5. 需要写入预览区时，必须调用 `write_learning_result`，并填写：
   - `style_skill_title`
   - `style_skill_body`
6. 禁止直接写入技能库，禁止生成独立本地分析文件，禁止绕过预览区。落盘动作必须由用户在界面中确认。

## 二、内部分析方法

通读正文时，在内部按以下维度标记典型片段。不要把大量原文搬进最终技能，只保留必要的短示例或概括。

| 维度 | 需要观察什么 |
|----|----|
| 情绪笔墨密度 | 同一角色有情绪和没情绪时，同类动作的文字量差异；情绪越强，感官、动作、停顿是否越展开 |
| 句式与段落节奏 | 短句、长句、断句、独句成段、连续短段、段尾翻转句、段落收束方式 |
| 叙述视角 | 第一人称或第三人称的受限程度；信息何时给、何时藏；内心独白与外部动作的比例 |
| 对话写法 | 对话标签用法、对白密度、留白方式、短-短-长节奏、权力关系如何通过话语暴露 |
| 动作写法 | 以动作代替心理描写、微动作承载情绪、身体反应替代情绪词 |
| 人物写法 | 是否弱化外貌，强化标签特征；人物是否通过选择、反应、沉默、拒绝来塑造 |
| 环境写法 | 环境何时出现、何时消失；是否作为情绪容器；同一地点在不同情绪下如何变化 |
| 转场与时间 | 时间跳跃是否果断；回忆、现实、过渡、场景切换如何完成 |
| 章节/分页收束 | 每节或每页最后一句如何制造余味、转折、悬念、情绪坠落或行动推进 |

核心判断原则：
- 写什么不重要，怎么写才重要。
- 结果必须能指导“分节写手”生成新正文，不要停留在抽象评价。
- 情绪决定笔墨密度：主角情绪强烈时，一秒钟的动作可以展开成一整段；主角心里没事时，一个动作可以一句话带过。
- 不要强行给无情绪场景注入细节。读者会追问角色为什么关注这些细节，从而造成误读。

## 三、最终预览区输出结构

调用 `write_learning_result` 时，`style_skill_body` 必须使用以下结构。可以根据样本实际情况调整措辞，但不得缺少核心栏目。

```markdown
# 文风学习：{作品或样本名称}分节写手技能

## 1. 一句话核心

用一句话概括这批样本文风最关键的写法原则。

## 2. 行文核心结构：情绪决定笔墨密度

### 2.1 有情绪时：场景展开写
规则：说明角色处于强烈情绪中时，哪些感官、动作、停顿、内心反应会被放大。

写法模板：
- 当主角处于【情绪状态】时，不急着推进事件，先写【感官/身体反应/微动作】。
- 一秒钟可以拆成【动作前停顿】、【动作本身】、【身体反应】、【情绪余波】。

短示例：
只给极短示例或概括，不要大段复制原文。

### 2.2 没情绪时：一句话带过
规则：说明日常动作、低情绪过渡、时间跳跃如何快速处理。

写法模板：
- 【时间/动作】一句带过，直接进入真正有情绪变化的场景。
- 不解释不重要的环境，不展开没有心理重量的动作。

### 2.3 笔墨密度速查表

| 情境 | 主角状态 | 笔墨密度 | 写法 |
|----|----|----|----|
| 被背叛 | 震惊/失控/麻木 | 高 | 放慢时间，写感官和身体反应 |
| 被羞辱 | 克制/压抑/反击前 | 高 | 用停顿、沉默、微动作承压 |
| 回忆涌上 | 被过去击中 | 高 | 现实动作触发回忆，回忆不解释过满 |
| 日常过渡 | 情绪平稳 | 低 | 一句话带过 |
| 时间跳跃 | 事件无关键变化 | 低 | 果断跨越，只保留结果 |
| 心死行动 | 冷静/决绝 | 中高 | 少情绪词，多动作和选择 |

## 3. 句式与段落节奏

写清样本常用句式、段落长度、断句方式、独句成段习惯。

必须包含：
- 常用句长：短句为主、长短交错或长句铺陈。
- 段落规则：什么时候一句一段，什么时候连续短段。
- 段尾规则：最后一句是否常用于翻转、补刀、落空、悬念或情绪坠落。
- 分页建议：每页结尾应留下一个情绪钩子或行动钩子。

## 4. 叙述视角与信息控制

写清叙述视角如何限制信息。

必须包含：
- 读者知道的事是否只跟随主角当下认知。
- 内心独白是否直接说破，还是通过动作和反应外化。
- 误会、隐瞒、反转信息通常如何释放。

## 5. 对话写法

写清对白密度、对话标签、留白、权力关系。

必须包含：
- 是否少用“他说/她说”等标签。
- 强势方如何说话：短、冷、命令、反问、沉默或压迫。
- 弱势方如何说话：解释、停顿、回避、反击或失语。
- 对话节奏是否呈现短-短-长、短句互撞、最后一句转向。
- 对话中哪些话不说出来，只让读者从沉默和动作里补全。

## 6. 动作与心理写法

写清如何用动作代替心理描写。

按情绪分类给出规则：
- 心碎/崩溃：写身体失控、感官错位、动作中断。
- 克制/忍耐：写停顿、攥紧、移开视线、吞回话。
- 绝望/放弃：写动作变慢、反应变少、语气变平。
- 决绝/力量感：写明确选择、干净动作、少解释。

## 7. 人物塑造写法

写清人物不是靠外貌堆砌，而是靠标签特征、选择和对照站住。

必须包含：
- 每个主要人物的标签特征：不要只写外貌，要写行为逻辑和情绪按钮。
- 人物如何通过选择塑造：他/她在关键处选择说什么、忍什么、放弃什么、拿回什么。
- 对立人物如何镜像：女主与女配、主角与反派、过去的自己与现在的自己有什么反差。

## 8. 环境与转场写法

写清环境和时间如何服务情绪。

必须包含：
- 环境只在情绪需要时出现；无情绪承载时快速消失。
- 环境作为情绪容器：同一地点可因心境不同呈现不同质感。
- 转场方式：时间跳跃、场景切换、回忆插入、章节承接如何完成。

## 9. 章节/分页收束方式

写清分节写手每一节、每一页应该怎么收。

至少给出 4 类收束模板：
- 情绪坠落式：最后一句让角色意识到更痛的事实。
- 行动推进式：最后一句落在一个明确动作上。
- 信息反转式：最后一句释放新信息，改变读者判断。
- 留白悬停式：最后一句停在沉默、未说出口的话或未完成动作上。

## 10. 分节写手执行模板

当你生成新正文时，按以下顺序执行：

1. 先判断本节核心情绪，不急着写事件。
2. 判断每个动作是否有情绪重量；有则展开，无则带过。
3. 对话只写能改变关系、暴露权力或推进情绪的句子。
4. 心理描写优先外化为动作、身体反应、沉默和选择。
5. 环境只在承载情绪时出现。
6. 每页结尾必须留下情绪钩子、行动钩子或信息钩子。

## 11. 反面典型速查表

| 写法维度 | 核心原则 | 反面典型 |
|----|----|----|
| 段落 | 跟随情绪切段 | 无论轻重都写成长段 |
| 视角 | 跟随主角认知 | 提前泄露主角不知道的信息 |
| 对话 | 暴露关系和压力 | 闲聊式解释设定 |
| 动作 | 动作承载心理 | 直接堆情绪词 |
| 人物 | 用选择塑造 | 只写外貌和人设标签 |
| 环境 | 为情绪服务 | 无意义铺景 |
| 时间 | 无关键变化就跳过 | 把过渡过程写满 |
| 笔墨密度 | 情绪越强越慢 | 无情绪动作也强行细写 |

## 12. 质量自检

- 是否能直接指导分节/分页写手生成新正文。
- 是否提炼了句式节奏、段落长度、叙述视角、对白密度、情绪词选择、动作描写、转场方式、章节/分页收束方式。
- 是否把“情绪决定笔墨密度”写成了可执行规则。
- 是否避免大段复制原文。
- 是否包含规则、模板、速查表和检查清单。
- 是否没有把分析结果写成本地文件，而是准备写入预览区等待用户确认落盘。
```

## 四、标题要求

`style_skill_title` 使用简洁标题，格式建议：

`{样本名称或作者名}文风学习：分节写手技能`

如果样本来源不止一篇，可使用：

## 五、写入前自检

调用 `write_learning_result` 之前，确认以下事项全部满足：

- 已经读取正文，不是只看概况。
- 最终内容是“写手技能”，不是单纯评论文章。
- 最终内容能指导生成新正文。
- 没有大段复制原文。
- 没有生成独立 `行文分析.md` 或其他本地分析文件。
- 最终只写入预览区，由用户确认后再落盘。

完成后，调用 `write_learning_result` 写入预览区。不要在普通回复中输出完整技能正文，除非工具不可用；如果工具不可用，再用简体中文说明原因，并提供可粘贴的 `style_skill_title` 和 `style_skill_body`。
