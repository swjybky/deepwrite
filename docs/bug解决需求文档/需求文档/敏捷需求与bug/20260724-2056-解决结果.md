## 问题概述

- 用户描述：串行调用子智能体时，先创建空白章节再写文，无法编辑尚未落盘的章节；建章审阅卡出现「版本冲突 / 正文目录已发生变化」。
- 影响模块：正文专家总控 / 子智能体工具链、待审阅提案、空白章节创建与正文写入接受链路。
- 触发路径：同一 run（含 `spawn_subagent` 串行）内 `create_expert_draft_sections` → `write_expert_draft_section` / replace；或同轮正文写入与建章提案交错接受。

## 根因分析

- 触发条件：同轮先建章再写文；或同轮先改已有章节正文再接受建章提案。
- 根因定位：
  1. `create_expert_draft_sections` 只记录待创建标题，未在 mutation overlay 物化可读写的临时章节（无 `section_id` / 文件），后续写工具无法定位目标。
  2. 目录 revision 把各章正文与人物状态内容 hash 一并计入，内容写入落盘后会误伤结构创建的乐观锁校验。
  3. auto-approve 在整轮结束后才落盘，同轮内写文必然早于真实章节存在；Renderer 也未承接 provisional 文件 mutation。
- 影响范围：短篇正文目录初始化 + 同轮连续写文；子智能体共享 overlay 场景；建章与正文写入的自动/手动审阅顺序。

## 修复内容

- 修改文件：
  - `packages/contracts/src/workspace.ts`：结构目录 revision 辅助函数；更新总控提示词。
  - `packages/contracts/src/session.ts`：建章 mutation 增加必填 `provisionalSectionId`。
  - `packages/pi-runtime-adapter/src/short-agent-tools.ts`：overlay 物化临时章节；目录/读/写可见；父子共享。
  - `packages/pi-runtime-adapter/src/index.ts`：事件类型同步。
  - `apps/desktop/src/renderer/src/App.vue`：结构 revision；provisional 写入 staging；接受时 id 映射；auto-approve 先建章后写文；拒绝/失败级联冲突。
  - `apps/desktop/src/renderer/src/composables/useAgentConversation.ts`：结构 revision 快照；提案深拷贝/恢复；`listEditProposals`。
  - `apps/desktop/src/renderer/src/types/conversation.ts`：提案字段扩展。
  - 相关单测与契约哈希更新。
- 核心改动：
  - **C**：`createExpertDraftDirectoryRevision` 仅 hash `id + title + wordCountRequirement`。
  - **A**：创建时分配 `pending:section:N`，写入 overlay；同轮可读可写；事件携带 provisional id；接受创建后 remap 待写提案到真实文档；队列优先接受建章。
- 兼容处理：会话恢复时若旧建章提案缺少 `provisionalSectionId`，补 `pending:section:legacy-*`，避免整卡提案被丢弃。
- 审查加固（2026-07-24）：
  1. 建章接受幂等，避免 `accepting` 重入重复建章。
  2. 部分建章失败时只冲突未落盘章节，已成功章节仍 remap。
  3. 目录 revision 与 runtime 快照一致，仅统计 body+人物状态成对齐全的章节。
